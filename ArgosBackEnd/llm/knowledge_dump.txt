[PROJECT: OzCab]
[GOAL: Digitize manufacturing audit process]
* MVP: Inspect, approve, reject manufactured parts.
* Future: HR, Finance modules. !Modularity critical.

[BIZ_LOGIC]
* [WORKFLOW: Main]
    * Client -> Service (Contract with start_date, end_date)
    * Service -> 1+ WorkInstruction (IT)
    * WorkInstruction -> 1 Part (model)
        * Defines inspection steps, criteria, defects.
        * inspection_rate_per_hour: Pieces/hour/inspector.
    * WorkInstruction -> 1+ InspectionReport (RI)
    * InspectionReport -> n InspectionDetail
    * InspectionDetail -> n Incident (if defect found)
    * Incident -> 1 Defect (from catalog)
* [USERS]
    * MVP Roles: Inspector, Manager, Admin.
    * Future: Client.

[TECH_STACK]
================================================================================

[BACKEND: /back]
* Framework: Express 5, ESM JavaScript
* DB: MySQL (ozcab_db) via mysql2 promise pool
* Auth: Firebase Admin SDK (session cookie verification)
* Pattern: Routes -> Handlers -> MySQL queries
* Response format: { success: true/false, data?: any, motive?: string }

[FRONTEND: /front]
* Framework: Next.js 15, App Router, TypeScript
* UI: shadcn-ui components, Tailwind CSS v4
* Auth: Firebase Client SDK + BFF pattern (API routes)
* State: React Context (UserContext for user/permissions)
* URL State: nuqs for query parameter management
* Data Mutations: Server Actions pattern

================================================================================
1. User logs in via Firebase Client SDK on frontend
2. Frontend calls /api/auth/handleLogin which creates session cookie
3. All protected API calls include session cookie
4. Backend validates via Firebase Admin SDK (verifySession middleware)
5. Frontend middleware fetches user permissions via /api/auth/getCurrentUser

[SERVER ACTIONS PATTERN]
Location: front/src/app/(protected)/<feature>/actions/<feature>.actions.ts

"use server";
import { cookies } from "next/headers";
import { revalidatePath } from "next/cache";

export async function updateSomething(id: number, data: Data) {
    const cookieStore = await cookies();
    const session = cookieStore.get('session')?.value;
    // Fetch to backend with session cookie
    const res = await fetch(`${EXPRESS_BASE_URL}/endpoint/${id}`, {
        method: 'PUT',
        headers: {
            'Content-Type': 'application/json',
            'Cookie': `session=${session}`,
        },
        body: JSON.stringify(data),
    });
    // Handle response
    revalidatePath('/path');  // Revalidate on success
    return { success: true };
}

[MODAL PATTERN (SHADCN DIALOG)]
* Use Dialog from @radix-ui/react-dialog (via src/components/ui/dialog.tsx)
* Control with open/onOpenChange props
* Fetch data with useEffect when modal opens (for update modals)
* Submit with useTransition for pending state
* Reset state when modal closes (useEffect on !open)
* Prevent close during submission: if (!newOpen && isPending) return;

Example structure:
export default function UpdateModal({ id, open, onOpenChange }) {
  const [isPending, startTransition] = useTransition();
  const [formData, setFormData] = useState({...});

  // Reset on close
  useEffect(() => { if (!open) resetForm(); }, [open]);

  // Fetch on open
  useEffect(() => { if (open && id) fetchData(); }, [open, id]);

  const handleOpenChange = (newOpen: boolean) => {
    if (!newOpen && isPending) return;
    onOpenChange(newOpen);
  };

  return <Dialog open={open} onOpenChange={handleOpenChange}>...</Dialog>;
}

[HYDRATION SAFETY]
* Use mounted state for client-only rendering
* Return null until mounted when using browser-only APIs

const [mounted, setMounted] = useState(false);
useEffect(() => { setMounted(true); }, []);
if (!mounted) return null;

[COMMON ISSUES & FIXES]
================================================================================

[HYDRATION MISMATCH]
* Cause: Server/client render different content
* Fix: Add mounted state check, return null until mounted

[MODAL NOT CLOSING / PAGE UNRESPONSIVE]
* Cause: onOpenChange not properly forwarding state
* Fix: Use handleOpenChange that passes boolean through to parent
* Reset state in useEffect when !open, not in handleClose

[SQL LIMIT/OFFSET ERROR]
* Error: "Incorrect arguments to mysqld_stmt_execute"
* Cause: Using ? placeholders for LIMIT/OFFSET
* Fix: Use integer interpolation after validation

[USER HAS NO PERMISSIONS]
* Check user_roles table has entry for user
* Add role: INSERT INTO user_roles (user_id, role_id) VALUES (user_id, role_id)
